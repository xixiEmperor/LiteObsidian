# LiteObsidian 当前迭代 CRUD 开发任务

## 目标
这轮只做 5 个功能，先把需求和验收对齐，不在这里写实现代码。

## 需求范围
1. 导入导出：导入 .md，导出单篇或多篇为 .md 或 .zip
2. 附件能力：相机或相册选图，文件落本地并回填到 Markdown
3. 组织能力：标签、置顶、最近编辑、收藏
4. 设置中心：字体、行宽、自动保存间隔、主题跟随系统
5. 编辑增强：Markdown 工具条、快捷插入模板、代码块语言选择

## 任务拆分

### A. 导入导出
- 前端
1. 新增导入入口，支持选择一个或多个 .md 文件
2. 新增导出入口，支持导出当前笔记和批量导出
3. 批量导出时提供 .zip 选项
- 原生
1. 提供文件选择能力，返回文件路径或内容
2. 提供写入导出目录能力
3. 提供 zip 打包能力
- 数据
1. 导入时做标题去重策略，默认保留原文并自动补后缀
2. 记录导入时间和来源
- 验收
1. 选中 .md 能导入为笔记
2. 单篇导出得到 .md 文件
3. 多篇导出得到 .zip，解压后内容完整

### B. 附件能力
- 前端
1. 编辑页新增插入图片入口
2. 支持拍照和相册两种来源
3. 插入后在正文回填 Markdown 图片语法
- 原生
1. 相机拍照并保存到应用私有目录
2. 相册选图并复制到应用私有目录
3. 返回可展示的本地 uri 或相对路径
- 数据
1. 新增附件表，记录 note_id、文件名、路径、大小、创建时间
2. 删除笔记时可配置是否联动删除附件
- 验收
1. 两种来源都可插入图片
2. 重启应用后图片仍可正常显示
3. 删除笔记后附件策略符合预期

### C. 组织能力
- 前端
1. 列表支持标签筛选
2. 列表支持置顶和收藏视图
3. 列表默认按最近编辑排序
- 原生
1. 提供标签增删改查接口
2. 提供置顶和收藏状态更新接口
- 数据
1. notes 表增加 is_pinned、is_favorite、last_opened_at
2. 新增 tags 和 note_tags 关系表
- 验收
1. 置顶永远在普通笔记前
2. 收藏可单独过滤
3. 标签筛选结果正确

### D. 设置中心
- 前端
1. 新增设置页入口和设置页
2. 提供字体大小、行宽、自动保存间隔、主题跟随系统开关
3. 设置修改后立即生效
- 原生
1. 提供读取系统主题能力
2. 提供设置持久化读写能力
- 数据
1. app_settings 表新增对应 key
- 验收
1. 重启后设置不丢失
2. 跟随系统开关后可随系统主题切换
3. 自动保存间隔修改后行为正确

### E. 编辑增强
- 前端
1. 新增 Markdown 工具条，覆盖标题、粗体、斜体、列表、引用、链接、图片、代码块
2. 新增快捷模板插入，比如会议纪要、学习笔记、日报
3. 代码块可选择语言并自动生成 fenced code block
- 原生
1. 无强依赖，优先在前端完成
- 验收
1. 工具条每个按钮都能正确插入或包裹文本
2. 模板插入后可直接编辑
3. 代码块语言选择后语法格式正确

## 联调接口草案
1. invoke('importMarkdown', { uris: string[] })
2. invoke('exportNotes', { ids: number[], format: 'md' | 'zip' })
3. invoke('pickImage', { source: 'camera' | 'gallery' })
4. invoke('upsertTags', { noteId: number, tags: string[] })
5. invoke('togglePin', { id: number, pinned: boolean })
6. invoke('toggleFavorite', { id: number, favorite: boolean })
7. invoke('getSettings', {})
8. invoke('saveSettings', { fontSize: number, lineWidth: number, autoSaveMs: number, followSystemTheme: boolean })

## 里程碑建议
1. 第一阶段：导入导出 + 设置中心
2. 第二阶段：附件能力 + 编辑增强
3. 第三阶段：标签、置顶、收藏、最近编辑

## 非本轮范围
1. 云同步
2. 多端账号体系
3. 在线协作
