# LiteObsidian 最小回归检查清单

这份清单用于每次打包前做快速手工回归，目标是用最少步骤覆盖核心链路。

## 执行前准备

1. 在 `web/` 执行 `pnpm install`
2. 在 `web/` 执行 `pnpm build:android`
3. 用 Android Studio 打开 `android/` 并安装到模拟器或真机

## 回归检查点

1. 启动应用可以正常进入首页，没有白屏和闪退
2. 首页在有数据时能看到列表，列表项可以点击进入详情页
3. 首页在无数据时可以看到空状态提示，并且可以点击新建按钮
4. 新建笔记后点击保存，返回列表可以立即看到新笔记
5. 进入已有笔记后修改正文，等待自动保存后有保存状态提示
6. 编辑中直接返回会出现离开确认，取消后仍留在当前页
7. 详情页切换到预览模式后，标题和列表和代码块可以正常渲染
8. 删除笔记时会先弹出二次确认，确认后列表里不再显示该笔记
9. 首页搜索框输入关键词后，列表可以按标题或正文过滤
10. 强制结束应用进程后重新打开，之前保存的笔记仍然存在

## 记录建议

每次回归可以记录三项信息，方便排查问题。

1. 执行日期和执行人
2. 失败的检查点编号
3. 复现路径和截图位置
